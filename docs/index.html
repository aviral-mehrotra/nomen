01 / The problem

Every screenshot you've ever taken is named after a timestamp.

Hundreds, maybe thousands of files in your Desktop or Downloads folder, each one indistinguishable from the next. Search is useless. You either rename each one by hand (tedious), don't bother (most common), or build elaborate Hazel rules (only power users).

Screenshot 2026-04-22 at 3.47.12 PM.png → login-flow-step-3.png

Nomen renames it before you close the prompt. Friction is in the right place — naming happens once, immediately, while context is still fresh.

02 / How it works

Cmd+Shift+4. Type a name. Done.

Zero behavior change at capture time. You still take screenshots the way you always have.

01 · Capture

Take a screenshot

Use Cmd+Shift+3, 4, or 5 — whatever you already use. The file saves as it always does.

02 · Name

A small floating panel appears

Type a meaningful name, optionally add tags, hit Enter. Esc skips and keeps the original name.

03 · Find

The file is renamed in place

Tags become Finder tags — searchable by Spotlight forever. Nothing leaves your Mac.

03 / Why Nomen

Quiet, fast, forgettable.

A tool you stop noticing because it just works. Built native to macOS — feels like part of the system, not an app pretending to be one.

All processing local

No screenshots ever leave your device. No analytics, no telemetry, no network calls. Your files, your privacy.

Doesn't steal focus

Keep typing in your current app. The prompt floats above without interrupting what you were doing.

Skip when you want

Press Esc and the file keeps its original name. Nomen never blocks your workflow.

Finder tags built in

Tag a screenshot once and find it forever via Spotlight, the Finder sidebar, or smart folders.

Multi-screenshot queue

Take a flurry of screenshots; Nomen prompts you for each one in order. No lost captures.

Sandboxed, signed, notarized

Apple-verified Developer ID. Hardened runtime. Tight macOS sandbox. Compliant with App Store rules even though it's distributed direct.
